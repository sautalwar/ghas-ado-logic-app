PROMPT 1: Initial Requirements
--------------------------------------------------------------------------------

Can we use a Logic App (or similar automation) to:

- Automatically create an ADO work item when a new GHAS vulnerability is detected
  From inside the GHAS vulnerability context
  Without a human needing to:
    - Go to Boards
    - Manually create a work item
    - Come back and link it to the vulnerability

- The work item should auto-populate:
    - Vulnerability title
    - Description
    - Severity
    - Repo / branch
    - File and line (if available)
    - Any GHAS metadata available via API

- Automatically link the created work item back to the GHAS vulnerability
    - So developers have full traceability
    - So security doesn't lose context

- (Nice-to-have, but explicitly discussed) Auto-close the work item when the
  vulnerability is resolved
    - If GHAS marks the vulnerability as fixed
    - The linked ADO work item should be automatically closed or transitioned
    - Customer said this is not a deal breaker, but would be "huge value"

Can you tell me how you will do this task?

HOW TO REPRODUCE
================================================================================

To rebuild this entire solution from scratch, paste the prompts above (1-10)
sequentially into a Copilot CLI session with:
  - Access to an Azure subscription (for Logic App deployment)
  - An ADO organization and project with a PAT
  - A GitHub account (for pushing code)
  - Playwright available (for screenshots)

The session will:
  1. Create all IaC, scripts, and documentation
  2. Deploy the Logic App to Azure
  3. Seed ADO with demo data
  4. Capture Azure Portal and ADO screenshots
  5. Test end-to-end (create + auto-close)
  6. Generate a PowerPoint presentation
  7. Push everything to GitHub
